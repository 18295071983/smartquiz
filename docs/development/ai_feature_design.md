# AI功能详细设计文档

## 1. 概述

### 1.1 文档目的
本文档详细描述智能题库应用中AI功能的设计方案，包括技术架构、功能模块、实现细节和集成方案，为开发团队提供完整的技术指导。

### 1.2 功能定位
AI功能作为智能题库应用的核心增强功能，旨在通过本地部署的大语言模型，为用户提供智能化的学习辅助服务，包括智能题目生成、题目解析、学习建议和多语言翻译等功能。

### 1.3 设计原则
- **本地化优先**：模型在设备本地运行，保护用户隐私
- **性能优化**：针对移动设备特性进行性能优化
- **模块化设计**：采用清晰的模块化架构，便于扩展和维护
- **用户体验**：确保AI功能响应迅速，界面流畅
- **离线可用**：支持无网络环境下正常使用

## 3. 功能模块设计

### 3.1 智能题目生成

#### 3.1.1 功能描述
基于用户指定的知识点和难度级别，自动生成符合要求的题目，包括题目文本、选项、正确答案和详细解释。

#### 3.1.2 技术实现
- **输入参数**：
  - 知识点（topic）：字符串
  - 题目数量（count）：整数
  - 难度级别（difficulty）：整数（1-3）

- **处理流程**：
  1. 构建提示词模板
  2. 调用AI服务生成题目
  3. 解析生成的题目格式
  4. 转换为Question对象

- **输出结果**：
  - List<Question>：生成的题目列表

#### 3.1.3 示例代码
```java
public List<Question> generateQuestions(String topic, int count, int difficulty) {
    // 构建提示词
    String prompt = String.format(
        "Generate %d %s difficulty questions about %s. Each question should include:\n" +
        "- Question text\n" +
        "- Multiple choice options (4 options)\n" +
        "- Correct answer\n" +
        "- Explanation\n",
        count, getDifficultyString(difficulty), topic
    );
    
    // 调用AI服务
    String response = aiService.generateSync(prompt, 1000);
    
    // 解析结果
    return parseGeneratedQuestions(response);
}
```

### 3.2 题目解析与解答

#### 3.2.1 功能描述
分析用户提供的题目，自动生成详细的解答，包括正确答案、详细解释、错误选项分析和相关知识点。

#### 3.2.2 技术实现
- **输入参数**：
  - Question对象：包含题目文本、选项等信息

- **处理流程**：
  1. 构建提示词，包含题目信息
  2. 调用AI服务生成解答
  3. 格式化解答结果

- **输出结果**：
  - String：详细的题目解答

#### 3.2.3 示例代码
```java
public String analyzeQuestion(Question question) {
    String prompt = String.format(
        "Analyze the following question and provide a detailed solution:\n" +
        "Question: %s\n" +
        "Options:\n" +
        "A. %s\n" +
        "B. %s\n" +
        "C. %s\n" +
        "D. %s\n" +
        "Please provide:\n" +
        "1. The correct answer\n" +
        "2. A detailed explanation\n" +
        "3. Explanations of why other options are incorrect\n" +
        "4. Key concepts tested",
        question.getQuestion(),
        question.getOptionA(),
        question.getOptionB(),
        question.getOptionC(),
        question.getOptionD()
    );
    
    return aiService.generateSync(prompt, 800);
}
```

### 3.3 学习辅助

#### 3.3.1 功能描述
基于用户的学习需求，提供个性化的学习计划、概念解释和考试备考建议。

#### 3.3.2 技术实现
- **子功能**：
  1. **个性化学习计划**：基于学科、难度和可用时间生成学习计划
  2. **概念解释**：用通俗易懂的语言解释复杂概念
  3. **考试备考建议**：基于考试类型和剩余时间提供备考策略

- **输入参数**：
  - 个性化学习计划：学科、难度、每周学习时间
  - 概念解释：概念名称、难度级别
  - 考试备考建议：考试类型、剩余天数

- **输出结果**：
  - String：详细的学习建议或计划

#### 3.3.3 示例代码
```java
public String getPersonalizedStudyPlan(String subject, int difficulty, int studyHoursPerWeek) {
    String prompt = String.format(
        "Create a personalized study plan for learning %s at %s difficulty level.\n" +
        "Study time available: %d hours per week.\n" +
        "Please include:\n" +
        "1. Weekly study schedule\n" +
        "2. Key topics to focus on\n" +
        "3. Recommended learning resources\n" +
        "4. Progress tracking methods",
        subject,
        getDifficultyString(difficulty),
        studyHoursPerWeek
    );
    
    return aiService.generateSync(prompt, 1000);
}
```

### 3.4 智能翻译

#### 3.4.1 功能描述
支持题目和学习材料的多语言翻译，包括文本翻译、题目翻译和语言检测。

#### 3.4.2 技术实现
- **子功能**：
  1. **文本翻译**：将普通文本翻译为目标语言
  2. **题目翻译**：保持题目格式的同时翻译内容
  3. **语言检测**：检测文本的语言类型

- **输入参数**：
  - 文本翻译：文本内容、目标语言
  - 题目翻译：题目各部分内容、目标语言
  - 语言检测：待检测文本

- **输出结果**：
  - String：翻译结果或检测到的语言

#### 3.4.3 示例代码
```java
public String translate(String text, String targetLanguage) {
    String prompt = String.format(
        "Translate the following text to %s.\n" +
        "Text: %s\n" +
        "Please provide only the translation without any additional explanation.",
        targetLanguage,
        text
    );
    
    return aiService.generateSync(prompt, 600);
}
```

## 4. 核心服务设计

### 4.1 模型管理服务 (ModelManager)

#### 4.1.1 功能描述
负责模型的下载、验证、缓存和管理，确保模型文件的正确存储和访问。

#### 4.1.2 核心方法
- `getModelPath(String modelName)`：获取模型文件路径
- `isModelAvailable(String modelName)`：检查模型是否可用
- `getModelSize(String modelName)`：获取模型文件大小
- `deleteModel(String modelName)`：删除模型文件

#### 4.1.3 实现细节
- 模型存储路径：`/data/data/com.oilquiz.app/files/ai_models/`
- 从assets复制模型到内部存储
- 模型文件验证机制

### 4.2 推理引擎服务 (LLMService)

#### 4.2.1 功能描述
负责模型的加载、推理执行和资源管理，是AI功能的核心执行单元。

#### 4.2.2 核心方法
- `initialize()`：初始化模型
- `initialize(String modelName)`：初始化指定模型
- `generate(String prompt)`：生成文本
- `generate(String prompt, int maxTokens)`：生成指定长度的文本
- `shutdown()`：关闭模型，释放资源

#### 4.2.3 实现细节
- 基于LlamaBridge JNI接口
- 支持模型参数配置
- 完善的错误处理机制

### 4.3 AI服务 (AIService)

#### 4.3.1 功能描述
作为AI功能的统一入口，提供异步任务处理和全局访问点。

#### 4.3.2 核心方法
- `getInstance(Context context)`：获取单例实例
- `initialize()`：初始化AI服务
- `generateAsync(String prompt)`：异步生成文本
- `generateSync(String prompt)`：同步生成文本
- `shutdown()`：关闭AI服务

#### 4.3.3 实现细节
- 单例模式设计
- 线程池管理
- 异步任务处理
- 全局错误处理
