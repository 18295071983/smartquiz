## 2. 开发标准与规范

### 2.1 代码风格
- 使用4个空格进行缩进，不使用制表符
- 每行代码长度不超过120个字符
- 方法体和代码块使用大括号包围，大括号放在行尾
- 不同逻辑块之间使用空行分隔
- 运算符两侧使用空格
- 逗号后使用空格

### 2.2 命名规范
- 包名：小写字母和点分隔，遵循反向域名规则 `com.oilquiz.app.模块名`
- 类名：大驼峰命名法（PascalCase），具有描述性
- 方法名：小驼峰命名法（camelCase），动词开头
- 变量名：小驼峰命名法（camelCase），具有描述性
- 常量：全大写，单词之间用下划线分隔
- 资源命名：
  - 布局文件：`activity_功能名.xml`、`fragment_功能名.xml`
  - 字符串资源：`功能名_描述`
  - 颜色资源：`颜色用途`
  - 图标资源：`ic_功能名`

### 2.3 架构设计
- 采用MVVM架构模式
- 分层设计：UI层、业务逻辑层、数据层
- 组件化开发，模块之间低耦合
- 包结构：
  ```
  com.oilquiz.app
  ├── ui/             # 界面相关
  │   ├── activity/   # Activity
  │   ├── fragment/   # Fragment
  │   ├── adapter/    # 适配器
  │   └── widget/     # 自定义控件
  ├── viewModel/      # 视图模型
  ├── repository/     # 数据仓库
  ├── model/          # 数据模型
  ├── database/       # 数据库相关
  ├── service/        # 服务
  ├── util/           # 工具类
  └── constant/       # 常量
  ```

### 2.4 数据库操作
- 使用Room Persistence Library
- 实体类使用`@Entity`注解
- DAO接口使用`@Dao`注解
- 数据库类使用`@Database`注解
- 所有数据库操作通过Repository层进行
- 复杂查询使用@Query注解
- 批量操作使用事务
- 避免在UI线程执行数据库操作

### 2.5 UI设计
- 使用ConstraintLayout作为根布局
- 避免嵌套过深的布局层次
- 使用dp作为尺寸单位
- 支持不同屏幕尺寸的适配
- 使用Material Design设计规范
- 定义统一的主题和样式
- 支持深色模式

### 2.6 错误处理
- 捕获具体的异常类型，避免捕获通用Exception
- 异常处理应包含适当的日志记录
- 向用户展示友好的错误信息
- 网络错误处理：处理网络连接失败、服务器返回错误等情况

### 2.7 性能优化
- 内存优化：避免内存泄漏，使用弱引用和软引用
- UI优化：使用RecyclerView替代ListView，实现视图缓存
- 数据库优化：使用索引加速查询，批量操作减少数据库访问次数
- 网络优化：使用OkHttp的缓存机制，实现请求合并和去重

### 2.8 安全规范
- 数据安全：用户密码加密存储，敏感数据传输使用HTTPS
- 权限管理：只请求必要的权限，使用运行时权限请求
- 代码安全：避免SQL注入，避免XSS攻击，验证用户输入

### 2.9 版本控制
- 使用Git进行版本控制
- 遵循Git Flow工作流程
- 分支命名规范：feature/功能名、bugfix/问题描述
- 提交信息清晰、简洁
- 代码审查：代码提交前进行自我审查，团队成员之间进行代码审查
- 版本号管理：遵循语义化版本号规范

## 3. 系统架构与模块划分

### 3.1 整体架构
- 分层架构：表示层、业务逻辑层、数据访问层、基础设施层
- 表示层：Activity/Fragment → Adapter → ViewModel → Repository → Data Source
- 业务逻辑层：Service → Manager → Utility → Handler → Validator
- 数据访问层：Room Database → DAO → Entity → Migration
- 基础设施层：Network → Storage → Security → Logging → Configuration

### 3.2 模块划分
- 用户模块：用户管理、认证授权
- 题目模块：题目管理、题库维护
- 答题模块：答题流程、成绩记录
- 学习计划模块：学习计划管理、进度跟踪
- 错题模块：错题收集、复习管理
- 笔记模块：学习笔记、知识整理
- OCR模块：文字识别、题目录入
- AI模块：智能辅助、个性化建议
- 导出模块：数据导出、格式转换
- 备份模块：数据备份、恢复

### 3.3 核心流程
- 答题流程：用户选择答题模式 → 加载题目数据 → 显示题目 → 用户作答 → 计算成绩 → 保存成绩记录 → 显示成绩分析 → 返回主界面
- 题目管理流程：进入题目管理 → 选择操作（添加、导入、编辑、删除） → 执行操作 → 更新题库 → 返回题目列表

## 4. API设计与使用

### 4.1 API设计原则
- RESTful设计：遵循REST架构风格
- 一致性：统一的命名规范和参数格式
- 安全性：实现适当的认证和授权机制
- 可扩展性：支持未来功能的扩展
- 性能优化：优化API响应时间和资源使用

### 4.2 API版本控制
- URL路径版本控制：使用`/api/v1/`、`/api/v2/`等路径前缀
- 向后兼容：新版本API应保持对旧版本的兼容
- 版本废弃：废弃的API应提供至少6个月的过渡期
- 版本通知：在响应头中添加API版本信息

### 4.3 认证与授权
- 认证方式：JWT认证、Bearer Token、刷新令牌
- 授权机制：基于角色的访问控制、资源级权限、权限检查
- 安全措施：HTTPS、Rate Limiting、输入验证、防止SQL注入

### 4.4 API接口
- 用户相关：注册、登录、获取用户信息、更新用户信息、删除用户
- 题目相关：获取题目列表、添加题目、获取题目详情、更新题目、删除题目
- 成绩相关：提交成绩、获取成绩列表、获取成绩详情、获取用户成绩
- 学习计划相关：创建学习计划、获取学习计划列表、更新学习计划、删除学习计划
- 错题相关：添加错题、获取错题列表、删除错题、获取用户错题
- 笔记相关：创建笔记、获取笔记列表、更新笔记、删除笔记
- AI相关：AI聊天、学习分析、生成题目、提交反馈
- 导出相关：导出题目、导出成绩、导出学习计划
- 系统相关：获取系统版本、获取系统状态、获取系统配置

### 4.5 请求与响应格式
- 请求格式：Content-Type: application/json，Authorization: Bearer {token}
- 响应格式：成功响应、分页响应、错误响应

## 5. 数据库设计与操作

### 5.1 数据库结构
- 数据库名称：smartquiz_database
- 数据库版本：18
- 使用技术：Room Persistence Library
- 核心表：
  - 用户表 (user)
  - 题目表 (question)
  - 成绩历史表 (score_history)
  - 学习计划表 (study_plan)
  - 错题表 (wrong_question)
  - 收藏题目表 (favorite_question)
  - 笔记表 (note)
  - 聊天历史表 (chat_history)
  - 日志表 (log_entry)

### 5.2 表关系
- 用户表与其他表之间为1:N关系
- 题目表与错题表、收藏题目表、笔记表之间为1:N关系
- 定义了完整的外键关系，确保数据完整性

### 5.3 操作流程
- 数据库初始化：调用AppDatabase.getDatabase(context)获取数据库实例
- 题目管理：添加、查询、更新、删除题目
- 成绩管理：记录、查询、删除成绩
- 用户管理：注册、登录、获取用户信息、更新用户信息、删除用户
- 学习计划：创建、更新、查询、删除计划
- 错题管理：添加、更新、查询、删除错题

### 5.4 优化策略
- 索引优化：为常用查询字段添加索引
- 查询优化：分页查询、按需查询、批量操作、缓存机制
- 事务管理：批量操作使用事务确保数据一致性
- 数据库迁移：开发环境和生产环境使用不同的迁移策略
- 备份与恢复：定期备份、手动备份和自动备份机制

## 6. Android适配规则

### 6.1 屏幕适配
- 屏幕尺寸分类：小屏幕、中屏幕、大屏幕、平板、电视
- 分辨率适配：密度无关像素(dp)、缩放无关像素(sp)、密度桶
- 布局适配：使用ConstraintLayout，避免硬编码尺寸，考虑横竖屏布局
- 多窗口适配：支持分屏模式，处理窗口大小变化

### 6.2 系统版本适配
- 最低SDK版本：API 26 (Android 8.0)
- 目标SDK版本：最新API
- 关键版本特性适配：Android 14+、Android 13、Android 12、Android 11、Android 10
- 兼容性实现：使用@TargetApi和@RequiresApi注解，使用Build.VERSION.SDK_INT进行版本判断

### 6.3 资源管理
- 资源目录结构：drawable、layout、values、mipmap等目录
- 资源命名规范：图片资源、布局资源、字符串资源、颜色资源、尺寸资源
- 资源优化：图片压缩、矢量图、资源懒加载、资源缓存
- 多语言支持：为每种语言创建values目录，使用字符串资源

### 6.4 性能优化
- 内存优化：使用SparseArray替代HashMap，及时释放不再使用的资源
- UI优化：使用RecyclerView替代ListView，实现视图复用
- 启动优化：减少启动时的初始化操作，使用App Startup库管理初始化
- 电池优化：减少后台运行时间，使用JobScheduler调度后台任务
